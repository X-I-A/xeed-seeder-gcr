Welcome to use Xeed - Http Module

Introduction

This Xeed Module can capture Xeed HTTP flow and publish the message of X-I protocol to Pubsub which is one of the entry point of Insight Module

Access Point (Post Method to send, Get Method to check):

  • https://..../publishers/pubsub/destinations/<project_id>/topics/<topic_id>

The http message must meet the following requirements:

  • Posted HTTP header must have at least one field with 'XEED-' prefix
  • 'Content-Type: text/plain' format is mandatory
  • 'Content-Encoding' should be 'gzip' when it exists
  • HTTP message body must be a JSON String with 'header' or 'body' component

Please check the project Xeed_slt for a realization example

Xeed - Insight - Agent (X-I-A Architecture) is a full scope data solution.
